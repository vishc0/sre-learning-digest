# Textbook Reading Guide — How to Use This Framework

## What Kind of Resource This Is

This is a textbook, not a reference manual. The chapters build on each other. Chapter 2 assumes you have read Chapter 1. Chapter 6 assumes you know the terms from Chapters 2–5. Reading chapters out of order produces confusion, not insight.

That said, once you have read the textbook through, the `reference/` folder gives you two one-page cheatsheets designed to be open on a second monitor during live work.

## How to Work Through the Capstone

The capstone is designed as a three-phase progression:

### Phase 1: Understand the system (read once, reference throughout)
[00 — Shopping Cart Reference System](capstone/00-shopping-cart-reference-system.md)

This is the ground truth for all exercises. It contains the service catalogue, dependency graph, journey map, current metrics, and known issues. Read it before starting any exercise set. Keep it open while working exercises.

### Phase 2: Work exercises without solutions

Work each exercise set after completing its corresponding textbook chapter:

| Exercise Set | After Chapter | Covers |
| --- | --- | --- |
| [01 — Structural Terms](capstone/01-exercise-set-structural-terms.md) | ch02 | CD, SCS, LBH |
| [02 — Blast Radius](capstone/02-exercise-set-blast-radius.md) | ch03 | DG, BRI, CC, FLI, DSA |
| [03 — Health Signals + Efficiency](capstone/03-exercise-set-health-signals-efficiency.md) | ch04 + ch05 | EBV, APR, MTBI, OCR, OL, RCR, TAF |
| [04 — Change Governance](capstone/04-exercise-set-change-governance.md) | ch06 | SCV, MRI, CSD, RV, SCW, EBR, DSS |
| [05 — Integrated Scenarios](capstone/05-exercise-set-scenarios.md) | ch07 + ch08 | Full framework applied |

**Rule**: work from first principles. Do not look at solutions until you have attempted every sub-question, even if your answer feels uncertain. The learning happens in the struggle, not the reading.

### Phase 3: Review solutions and identify gaps

Solutions are in `capstone/solutions/`. For each exercise you got wrong: go back to the chapter, find the formula, and re-work the exercise from scratch. A wrong answer that you fully understand after review is more valuable than a correct answer you got by luck.

## How the Chapters Build on Each Other

```text
ch01: The Problem (no formulae — conceptual foundation)
  ↓
ch02: Structural Cluster — CD, SCS, LBH
  (describes the shape of the system)
  ↓
ch03: Blast Radius Cluster — DG, BRI, CC, FLI, DSA
  (describes what happens when the system fails)
  ↓

  (describes how well we can see what's happening)

ch05: Efficiency Cluster — OL, RCR, TAF
  (describes whether our work is moving the right needles)
  ↓
ch06: Change Governance — SCV, MRI, CSD, RV, SCW, EBR, DSS
  (describes how to change the system safely)
  ↓
ch07: Lifecycle Cluster — GEI, RST, PCC, SLTD, SSE, PPC, RDR, SCI, EDFD
  (describes how to handle growth, transition, and decommission)
  ↓

  (combines everything into four strategic indicators)
  ↓
ch09: Daily Practice
  (applies the full framework to the five activities SREs do every day)
```

## The Raw Material Folder

The `raw/` folder contains the original research documents and scenario deep-dives written before the textbook restructuring. These are preserved because they contain:

- `raw/research/` — detailed architecture documents, user journey analysis, and 6-layer technical breakdowns of the shopping cart system
- `raw/scenarios/` — 9 textbook-format scenario documents (greenfield service addition, feature flag launch, Kafka upgrade, Black Friday surge, EKS migration, zero-day patch, incident-driven architecture change, service decommission, third-party API deprecation)
- `raw/05-industry-standards-formulae.md` — Google SRE, DORA, AWS Well-Architected, ITIL, Chaos Engineering, and NIST expressed as formulae using the 25 coined terms
- `raw/06-daily-sre-activities.md` — the original daily practice handbook

The textbook chapters supersede these for learning purposes. The scenario documents in `raw/scenarios/` are complementary deep-dives for scenario-01 through scenario-09 — after completing ch07 and the lifecycle exercise set, reading the relevant scenario document adds operational depth.

## One Number to Know Before You Start

**SRMI = 0.179 — that is the current reliability maturity score of the shopping cart system.**

The target is > 5.0. The current score is 3.2% of target.

This single number explains everything this framework is trying to address: a system that declares 99.95% checkout SLA but delivers 99.62% actual availability, has 45.7% observability coverage, 31% alert precision, and a cascade coefficient that makes the checkout journey mathematically unsafe.

The framework gives you the tools to change that number — and to prove, with mathematics, that it is changing.
