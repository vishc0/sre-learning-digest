# Architecture Component Classification Worksheet

## Instructions
Complete one worksheet per component being classified. A component is any independently deployable unit. Fill out every section in order — the classification questions in Section 2 drive the classification decision in Section 3. Do not skip to the answer.

Classification disputes must be documented in Section 3. Disputed classifications are escalated to the Architect Reviewer before being entered into the layer-ownership-matrix.

## Section 2: Classification Questions

Answer each question with **Yes** or **No**, followed by one sentence of rationale. Do not skip any question — all seven must be answered even if the answer is obviously No.

**Q1. Does this component store data persistently?**
- Answer: [Yes / No]
- Rationale: [one sentence — e.g., "This component writes to a local PostgreSQL database and retains data across restarts."]
- Layer Implication: If Yes → candidate for Layer 1 (Persistent Storage) or Layer 2 (Storage Abstraction / Cache)

**Q2. Does this component exclusively expose data from a backing store via API with no business logic?**
- Answer: [Yes / No]
- Rationale: [one sentence — e.g., "This service is a thin REST wrapper over the customer_db with no computed fields or decision logic."]
- Layer Implication: If Yes → candidate for Layer 3 (Data Services)

**Q3. Does this component route, buffer, or translate messages asynchronously between systems?**
- Answer: [Yes / No]
- Rationale: [one sentence — e.g., "This component consumes from a Kafka topic, transforms the payload, and publishes to a RabbitMQ exchange."]
- Layer Implication: If Yes → candidate for Layer 4 (Middleware / Integration)

**Q4. Does this component implement business logic, decisions, or orchestration?**
- Answer: [Yes / No]
- Rationale: [one sentence — e.g., "This service validates order eligibility, applies promotional pricing, and calls payment and inventory services."]
- Layer Implication: If Yes → candidate for Layer 5 (Application Services)

**Q5. Does this component route, authenticate, rate-limit, or terminate external traffic at the network edge?**
- Answer: [Yes / No]
- Rationale: [one sentence — e.g., "This is the Kong API gateway — it handles JWT validation, rate limiting, and TLS termination for all external API consumers."]
- Layer Implication: If Yes → candidate for Layer 6 (API Gateway / Edge)

**Q6. Does this component render UI or deliver content directly to end users in a browser or mobile client?**
- Answer: [Yes / No]
- Rationale: [one sentence — e.g., "This is the React SPA bundle served from CloudFront — it renders all customer-facing UI."]
- Layer Implication: If Yes → candidate for Layer 7 (Presentation / UI)

**Q7. Does this component have characteristics of more than one layer?**
- Answer: [Yes / No]
- Rationale: [one sentence describing the ambiguity — e.g., "This service caches data in-process (L2 behavior) but also applies some filtering logic (L5 behavior)."]
- Resolution: [how you resolved the ambiguity — assign to the layer that represents the PRIMARY function]
