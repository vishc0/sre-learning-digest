# SLSA Level 2 Compliance Mapping
## DevSecOps Pipeline — Portfolio Project

SLSA (Supply-chain Levels for Software Artifacts) is a security framework for
software supply chains. Think of it as SOC2 compliance levels, but for build
pipelines. Level 0 = no guarantees. Level 4 = hermetic, reproducible, fully verified.

**Target**: SLSA Level 2 (Industry standard for cloud-native production workloads)
**Status**: Compliant

---

## SLSA Level Requirements and Pipeline Mapping

### Level 1 Requirements (Foundation)

| Requirement | Description | Pipeline Implementation | Status |
|---|---|---|---|
| Scripted build | Build is fully scripted, no manual steps | GitHub Actions YAML — fully automated | PASS |
| Provenance exists | Build produces a provenance document | SLSA GitHub Generator produces attestation | PASS |
| Provenance complete | Provenance includes all required fields | Generator includes: builder, source, build steps, materials | PASS |

### Level 2 Requirements (Hosted Build Service)

| Requirement | Description | Pipeline Implementation | Status |
|---|---|---|---|
| Version controlled | Build config is in version control | `.github/workflows/devsecops-pipeline.yml` in repo | PASS |
| Authenticated provenance | Provenance is signed | `slsa-github-generator` signs with Sigstore/Fulcio | PASS |
| Service-generated provenance | Build SERVICE generates provenance (not build script) | `slsa-framework/slsa-github-generator` runs as separate workflow | PASS |
| Hosted build platform | Build runs on a hosted, managed platform | GitHub Actions hosted runners (ubuntu-latest) | PASS |
| No self-hosted runners | Cannot use self-hosted runners for Level 2 | Confirmed: all jobs use `runs-on: ubuntu-latest` | PASS |

### Level 3 Requirements (Gap Analysis — Not Yet Implemented)

| Requirement | Description | Gap | Effort to Close |
|---|---|---|---|
| Hardened platform | Build platform has specific security controls | GitHub Actions hardened runner required (or Tekton Chains) | ~2 weeks |
| Non-falsifiable provenance | Provenance cannot be forged by build script | Requires dedicated provenance generator with isolated signing | Included in hardened platform |
| Ephemeral environment | Build environment is isolated per-build | GitHub Actions already provides this — PARTIAL CREDIT | Already met |

### Level 4 Requirements (Gap Analysis — Production Critical Only)

| Requirement | Description | Gap | Effort |
|---|---|---|---|
| Hermetic build | Build has no network access during compilation | Requires fully air-gapped build environment or Bazel hermetic | 4-6 weeks |
| Reproducible build | Same input = identical binary, deterministic | Requires reproducible build toolchain (very hard for Docker/Python) | 6-12 weeks |
| Two-person review | All changes require 2 reviewer approval | CODEOWNERS + branch protection = PARTIAL CREDIT | Low effort |

## Provenance Document Structure

The SLSA provenance attestation generated by this pipeline contains:

```json
{
  "predicateType": "https://slsa.dev/provenance/v0.2",
  "predicate": {
    "builder": {
      "id": "https://github.com/slsa-framework/slsa-github-generator/.github/workflows/generator_container_slsa3.yml@refs/tags/v2.0.0"
    },
    "buildType": "https://github.com/slsa-framework/slsa-github-generator/container@v1",
    "invocation": {
      "configSource": {
        "uri": "git+https://github.com/<org>/<repo>@refs/heads/main",
        "digest": { "sha1": "<commit-sha>" },
        "entryPoint": ".github/workflows/devsecops-pipeline.yml"
      }
    },
    "materials": [{
      "uri": "git+https://github.com/<org>/<repo>@refs/heads/main",
      "digest": { "sha1": "<commit-sha>" }
    }]
  }
}
```

**What this proves**:
- The EXACT source commit that produced this image
- The EXACT build workflow that ran
- The identity of the build system (GitHub Actions, not a compromised developer laptop)
- Tamper-evidence: the attestation is signed; any modification invalidates the signature

## Verification Instructions

After pulling an image from the registry, any consumer can verify the full supply chain:

```bash
# 1. Verify the image signature
cosign verify \
  --certificate-identity-regexp="https://github.com/<YOUR_ORG>/<YOUR_REPO>/.github/workflows/devsecops-pipeline.yml" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  ghcr.io/<YOUR_ORG>/<YOUR_REPO>@sha256:<DIGEST>

# 2. Retrieve and verify the SLSA provenance attestation
cosign verify-attestation \
  --type slsaprovenance \
  --certificate-identity-regexp="https://github.com/slsa-framework/slsa-github-generator" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  ghcr.io/<YOUR_ORG>/<YOUR_REPO>@sha256:<DIGEST>

# 3. Retrieve the SBOM attestation
cosign verify-attestation \
  --type cyclonedx \
  --certificate-identity-regexp="https://github.com/<YOUR_ORG>/<YOUR_REPO>" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  ghcr.io/<YOUR_ORG>/<YOUR_REPO>@sha256:<DIGEST> | jq '.payload' | base64 -d | jq .
```
