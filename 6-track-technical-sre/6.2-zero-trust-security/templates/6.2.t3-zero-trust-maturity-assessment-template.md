## Maturity Level Definitions

| Level | Description |
|-------|-------------|
| **Traditional** | Manual processes; static policies; siloed tools; no automation |
| **Advanced** | Partially automated; some integration across tools; risk-based decisions for some use cases |
| **Optimal** | Fully automated; dynamically updated policies; integrated across all pillars; continuous validation |
