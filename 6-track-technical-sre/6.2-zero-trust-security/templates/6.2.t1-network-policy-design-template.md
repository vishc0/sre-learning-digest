## Default Deny Configuration

| Question | Answer | Rationale |
|----------|--------|-----------|
| Apply default-deny ingress? | Y / N | |
| Apply default-deny egress? | Y / N | |
| Namespace-scoped or workload-scoped? | Namespace / Workload | |

**Default deny policy (include or exclude — do not leave blank):**

If default-deny ingress is Y:
```yaml
# Default deny ingress — [namespace-name]
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-ingress
  namespace: [namespace-name]
spec:
  podSelector: {}
  policyTypes:
  - Ingress
```

If default-deny egress is Y:
```yaml
# Default deny egress — [namespace-name]
# NOTE: DNS egress (port 53) must be explicitly allowed for pods that do DNS lookups
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-egress
  namespace: [namespace-name]
spec:
  podSelector: {}
  policyTypes:
  - Egress
```

## Ingress Rules

Document each explicit ingress allow rule. Every rule requires a business justification.

| # | Source Namespace | Source Pod Selector | Port | Protocol | Justification | Approved By |
|---|-----------------|--------------------|----|---------|--------------|------------|
| 1 | | | | TCP/UDP | | |
| 2 | | | | TCP/UDP | | |
| 3 | | | | TCP/UDP | | |
| 4 | | | | TCP/UDP | | |

**Notes on ingress rules**:
- NetworkPolicy rules are evaluated as OR (any matching rule allows the traffic)
- A single policy with multiple rules is equivalent to multiple policies
- Port can be a number (8080) or a named port (`http`)
- Cross-namespace selectors require `namespaceSelector` — verify namespaces have the required labels

## Egress Rules

Document each explicit egress allow rule. Egress rules are often missed — every external dependency must be listed.

| # | Destination Namespace | Destination Pod Selector / IP Block | Port | Protocol | Justification | Approved By |
|---|----------------------|-----------------------------------|------|---------|--------------|------------|
| 1 | kube-system | kube-dns | 53 | UDP/TCP | DNS resolution | Required for all pods |
| 2 | | | | | | |
| 3 | | | | | | |
| 4 | | | | | | |
| 5 | External: _____ CIDR | | | | | |

**Notes on egress rules**:
- DNS egress (UDP/TCP port 53 to kube-system kube-dns pods) must be explicitly allowed if default-deny egress is applied — forgetting this breaks all DNS-dependent functionality
- External IP blocks (for third-party API calls) use `ipBlock` with CIDR notation — verify the CIDR is current and as narrow as possible
- Kubernetes NetworkPolicy does not support FQDN-based egress — for DNS-named external services, either use Cilium's FQDN network policy or route through a controlled egress gateway

## YAML Policy — Complete NetworkPolicy Resource

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: [policy-name]
  namespace: [namespace-name]
  labels:
    managed-by: gitops
    policy-owner: [team-name]
    last-reviewed: "[YYYY-MM-DD]"
  annotations:
    description: "[Brief description of what this policy does and why]"
    ticket: "[Change record or Jira ticket]"
spec:
  podSelector:
    matchLabels:
      app: [workload-label-value]
  policyTypes:
  - Ingress
  - Egress
  ingress:
  # Rule 1: Allow from [source-description]
  - from:
    - namespaceSelector:
        matchLabels:
          kubernetes.io/metadata.name: [source-namespace]
      podSelector:
        matchLabels:
          app: [source-app-label]
    ports:
    - protocol: TCP
      port: [port-number]
  # Add additional ingress rules below
  egress:
  # Rule 1: Allow DNS (required for all pods with default-deny egress)
  - to:
    - namespaceSelector:
        matchLabels:
          kubernetes.io/metadata.name: kube-system
      podSelector:
        matchLabels:
          k8s-app: kube-dns
    ports:
    - protocol: UDP
      port: 53
    - protocol: TCP
      port: 53
  # Rule 2: Allow to [destination-description]
  - to:
    - namespaceSelector:
        matchLabels:
          kubernetes.io/metadata.name: [destination-namespace]
      podSelector:
        matchLabels:
          app: [destination-app-label]
    ports:
    - protocol: TCP
      port: [port-number]
  # Add additional egress rules below
```

## Pre-Deployment Testing Checklist

Complete in staging before applying to production.

**Functional validation:**
- [ ] All legitimate ingress traffic flows (test each ingress rule explicitly)
- [ ] All legitimate egress traffic flows (including DNS — test with `nslookup` from inside pod)
- [ ] Denied traffic is actually denied (test a connection that should be blocked and verify it is)
- [ ] Pod startup succeeds after policy application (some pods require egress on startup for dependency checks)

**Policy syntax validation:**
```bash
# Dry-run apply to validate YAML is accepted
kubectl apply --dry-run=server -f network-policy.yaml

# Apply in staging
kubectl apply -f network-policy.yaml -n [staging-namespace]

# Verify policy was accepted
kubectl get networkpolicy -n [staging-namespace]
kubectl describe networkpolicy [policy-name] -n [staging-namespace]
```

**Connectivity tests from staging:**
```bash
# Test allowed ingress (from source pod to this workload)
kubectl exec -n [source-namespace] [source-pod] -- \
  curl -v http://[workload-service].[namespace].svc.cluster.local:[port]/health
# Expected: HTTP 200

# Test blocked ingress (from a pod that should NOT have access)
kubectl exec -n [unauthorized-namespace] [test-pod] -- \
  curl --connect-timeout 5 http://[workload-service].[namespace].svc.cluster.local:[port]/health
# Expected: Connection timeout (blocked by NetworkPolicy)
```

- [ ] Blocked connections (actually blocked): PASS
- [ ] DNS resolution inside pod: PASS
