# SIEM/SOC Integration for SRE — Training Framework

## What This Framework Is

This framework teaches an SRE Manager — specifically one targeting a Director or VP of Platform Engineering role — how to articulate and operate at the interface between Site Reliability Engineering and Security Operations (SOC). It assumes Splunk as the primary observability and SIEM platform, because that is where most of the real work happens.

This is not a security certification study guide. It is not a Splunk training manual. It is a decision-making framework for a platform leader who needs to own the SRE-SOC interface, not just hand off at the boundary.

---

## Why This Is a Director Differentiator

Most SRE leaders can talk about SLOs, error budgets, and incident response. Fewer can articulate how a reliability platform and a security platform share infrastructure, data, and workflows. Almost none have a working model for how the two teams coordinate during a security-flavored reliability incident.

A Director who can answer the question "How do you think about the SRE-SOC interface?" with a coherent answer covering org model, data strategy, and handoff protocol is genuinely rare. Most candidates give one of two wrong answers: "That's a security team problem" or "We use the same Splunk so we're integrated." Neither answer reflects operational reality.

This framework gives you the third answer — the one that demonstrates you have actually operated at this intersection.

---

## Who This Framework Is For

- SRE Managers and Senior SREs moving toward Director/VP roles
- Platform engineers who own Splunk as a shared observability and SIEM platform
- Anyone preparing for interviews where security engineering posture of the platform team will be evaluated

Assumed background: Kubernetes/EKS operations, Splunk (including MLTK), incident command experience, Splunk MART framework familiarity.

## How to Use This Framework

**For interview preparation**: Read ch01, ch05, and ch08 first. These three chapters give you the strategic vocabulary. Then read the rest in order.

**For operational preparation**: Start with the playbooks. Then read ch03 and ch04 for the platform context. Use the reference cards during actual investigations.

**For Director-level framing**: Every chapter ends with a "Director frame" section. These are the talking points — the sentences you use in a panel interview, a staff meeting, or a board update. Read only those sections if time is short.
