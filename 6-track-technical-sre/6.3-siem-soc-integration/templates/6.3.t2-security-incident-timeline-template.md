## Phase Tracking

Track which phases are complete. Phases are concurrent — SRE and SOC phases run in parallel.

| Phase | SRE Owner | Status | Start Time | End Time |
|-------|-----------|--------|-----------|---------|
| Detection | [Name] | [Not Started / In Progress / Complete] | | |
| Containment (SRE) | [Name] | | | |
| Evidence Collection (SOC) | [Name] | | | |
| Scope Assessment (SOC) | [Name] | | | |
| Eradication | [SOC lead, SRE execute] | | | |
| Recovery (SRE) | [Name] | | | |
| Post-Incident | Both | | | |

**Phase definitions:**

- **Detection:** First signal observed to incident declared
- **Containment (SRE):** Service blast radius contained, traffic rerouted
- **Evidence Collection (SOC):** Forensic state captured before remediation actions
- **Scope Assessment (SOC):** Determination of all affected systems and data
- **Eradication:** Attacker access removed, compromised systems rebuilt
- **Recovery (SRE):** Service fully restored to normal capacity and health
- **Post-Incident:** RCA completed (SRE), forensic report completed (SOC)

## Shared Incident Timeline

Add entries in chronological order. Include ALL significant actions by both SRE and SOC. If an action is taken that might destroy forensic evidence, mark it clearly.

| # | Timestamp (UTC) | Actor | Actor Role | Action Taken | Evidence / Artifact | Notes |
|---|----------------|-------|-----------|-------------|-------------------|-------|
| 1 | YYYY-MM-DD HH:MM | [Name] | [SRE / SOC / System] | [What happened or was done] | [Link to Splunk search / S3 object / screenshot] | |
| 2 | | | | | | |
| 3 | | | | | | **[EVIDENCE RISK: this action may destroy forensic state — SOC approved at HH:MM]** |
| 4 | | | | | | |

**Action Categories:**
- `DETECT` — a signal was observed or an alert fired
- `ESCALATE` — a notification was sent to another team or individual
- `CONTAIN` — an action was taken to limit blast radius or scope
- `PRESERVE` — an artifact was captured for forensic use
- `RESTORE` — a service or system was returned to operational state
- `INVESTIGATE` — an investigation step was executed
- `COMMUNICATE` — an external or internal communication was sent
- `DECIDE` — a decision was made that changed the incident direction
- `ERADICATE` — attacker access was revoked or cleaned up
