# Reference: FinOps Maturity Assessment

**Purpose**: Self-assessment tool for determining FinOps maturity level and identifying the highest-value next steps. Use quarterly to track program progress.  
**Framework**: Based on FinOps Foundation Crawl/Walk/Run model, adapted for SRE-led platform organizations.  
**Scoring**: For each criterion, mark Current State as: ✅ Met | 🟡 Partial | ❌ Not Met

---

## How to Use This Assessment

1. Score every criterion honestly — partial credit is fine
2. Sum the scores by domain (Visibility, Optimization, Governance, Culture)
3. Determine your maturity level from the overall score
4. Use the "highest-value next step" column to build your roadmap

**Scoring**: ✅ = 2 points | 🟡 = 1 point | ❌ = 0 points

## Recommended Starting Actions by Maturity

### If You Are at Crawl

Priority order (highest ROI per effort):
1. **Tag coverage** — implement required-tags AWS Config rule. This unlocks everything else.
2. **Cost Anomaly Detection** — enable and configure Slack alerts. Zero cost; immediate protection.
3. **Monthly review** — schedule the first meeting and commit to the cadence. No tools required.
4. **Savings Plan baseline** — run Coverage Report; buy to 50% coverage. Immediate discount.
5. **Kubecost** — deploy in the EKS cluster. Unlocks namespace-level visibility.

### If You Are at Walk

Priority order:
1. **Right-sizing campaign** — largest single optimization lever. Quarterly cadence.
2. **Spot migration** — 60–80% discount on eligible compute. Implement mixed node groups.
3. **Non-prod scheduling** — kube-downscaler. Fast win, low risk.
4. **Savings Plan to 70%** — close the coverage gap.
5. **Cost SLOs** — define cost per unit; add to team OKRs.

### If You Are at Run

Priority order:
1. **Unit economics refinement** — improve model accuracy; automate data collection.
2. **Cost in architecture review** — formalize the process for new services.
3. **3-year RI for proven stable infrastructure** — Cassandra, message brokers, proven instance families.
4. **Topology-aware routing** — if not already done; for clusters with high inter-AZ traffic.
5. **FinOps OKRs** — ensure cost efficiency is a first-class organizational metric.
