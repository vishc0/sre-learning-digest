## Enterprise Support Tier and TCO

| Vendor | Support Model | SLA options | TCO Consideration |
|--------|--------------|-------------|------------------|
| **NetApp** | Enterprise Plus (24x7, 4h/2h onsite), Premier/Core | Up to 15-min response | Licensing is complex — NAS/SAN licenses, SnapMirror licenses, FlexClone separate. FAS to AFF migration is disruptive |
| **Pure Storage** | Evergreen (subscription includes controller refreshes and software) | 15-min phone response (Evergreen//Forever) | Higher upfront, lower lifecycle cost. No forklift upgrades. Software is included. Ops simplicity reduces staff cost |
| **Dell EMC** | ProSupport Plus, Mission Critical | 4h parts, 2h onsite | Broad product line — negotiate at volume. PowerMax licensing is complex. Strong for organizations with existing Dell server/networking relationships |
| **IBM** | Platinum (24x7, 4h), Gold, Essential | Up to 2h | FlashSystem value depends on IBM ecosystem usage (mainframe, Spectrum Scale). Standalone IBM storage without mainframe is harder to justify vs alternatives |
| **Ceph** | Red Hat Ceph Storage (RHCS) for commercial support, or community | RHCS SLA depends on subscription | Lowest $/TB but operational cost is highest. Requires Ceph expertise in-house or Red Hat consulting. Best TCO only if staffed appropriately |
| **AWS** | AWS Support (Developer/Business/Enterprise) | Enterprise: TAM + 15-min response | No hardware lifecycle, no DC space/power. Pay for IOPS/throughput separately. TCO vs on-prem depends heavily on egress and data volume. Use AWS Pricing Calculator |

## Director-Level Considerations for T-Mobile Enterprise Telco

| Vendor | What matters in a T-Mobile-class telco context |
|--------|-----------------------------------------------|
| **NetApp ONTAP** | MetroCluster synchronous replication for DC-to-DC DR is a genuine differentiator for mission-critical telecom billing and OSS/BSS data. FabricPool tiering to on-prem object store or S3 is a real cost lever. The complexity is manageable if you have NetApp-trained staff. |
| **Pure Storage** | The Evergreen subscription model eliminates the forklift upgrade cycle — at telco scale, every major storage refresh is a multi-year project risk. Pure's latency guarantee in contract is useful for SLO commitments to internal customers. FlashBlade for unstructured data (CDR, log, media) at scale. |
| **Dell EMC PowerMax** | Six nines availability and SRDF synchronous replication are used in financial services and large telcos for core network function databases. If T-Mobile runs mainframe workloads, PowerMax is likely already in the environment. |
| **Dell EMC PowerStore** | Good mid-range unified platform. Reasonable cost, good data services. Strong if you have existing Dell relationships for server/networking — consolidates the vendor relationship. |
| **IBM FlashSystem** | Relevant if mainframe is in scope (BSS systems on IBM mainframe are common in legacy telcos). IBM Spectrum Scale for HPC/ML at scale — if you build an AI/ML platform. Otherwise, harder to justify vs Pure or NetApp. |
| **Ceph (Rook/RHCS)** | The cost argument is compelling at telco data volumes. A petabyte of Ceph on commodity hardware is dramatically cheaper than a petabyte of NetApp or Pure. The operational investment is real — but a telco with a platform engineering team of the right size can absorb it. On-prem object storage (Ceph RGW as S3 endpoint) for CDR retention, Splunk SmartStore, and backup is where Ceph wins at telco scale. |
| **AWS** | EKS workloads — EBS and EFS are the right answers for cloud-native. S3 for anything that tolerates object storage latency. The hybrid reality at T-Mobile means on-prem storage remains the dominant tier for network-facing workloads; AWS is the right answer for platform engineering (Kubernetes, CI/CD, observability) and data analytics workloads. |

## Vendor Selection Decision Guide

| If your primary need is... | Start here |
|---------------------------|-----------|
| Highest performance for OLTP databases (Oracle, SAP HANA) | Pure FlashArray or NetApp AFF A-series |
| Unified NAS + SAN from one platform | NetApp ONTAP AFF |
| Synchronous DC-to-DC DR on-prem | NetApp MetroCluster or Dell PowerMax SRDF |
| Mainframe + SAN integration | Dell EMC PowerMax or IBM FlashSystem |
| Kubernetes on bare metal, multi-protocol | Ceph with Rook |
| Simplest operational model for all-flash SAN | Pure Storage (Purity UI, Evergreen subscription) |
| EKS-native storage | AWS EBS (block) + EFS (shared) + S3 (object) |
| Cost-optimized on-prem object storage at petabyte scale | Ceph RADOS Gateway or MinIO |
| ML/HPC parallel filesystem on-prem | IBM Spectrum Scale or Lustre (on-prem or FSx Lustre) |
| Vendor-neutral, lowest $/TB on-prem block | Ceph with budget hardware |
