### Director-level themes that recur across all chapters

**1. Storage tier routing is a cost and performance decision, not a familiarity decision.**  
Most organizations over-NAS and under-object. Every workload should sit on the cheapest tier that meets its SLO. The Director job is to create the routing policy and enforce it.

**2. Kubernetes storage is where stateful workloads fail.**  
EBS, EFS, Ceph, Longhorn — each has operational implications. Know the failure modes before the failure happens. The choice of managed vs self-hosted has a 3x operational cost difference.

**3. SAN justification requires a business case.**  
Fibre Channel SAN at $$/TB is only justified by workload IOPS or latency requirements that nothing else can meet. Enterprise SAN bought out of habit is budget waste that a Director should flag.

**4. Object storage is the long-game cost lever.**  
Lifecycle policies, versioning, and S3-compatible self-hosted options (MinIO, Ceph RGW) are where you recover budget at scale. Telco log retention at T-Mobile scale is an object storage problem, not a NAS problem.

**5. Capacity planning is a leadership responsibility.**  
Running to 95% full is a process failure, not a technical failure. The Director owns the signal that procurement happens before the crisis, not during it.

## T-Mobile Context Notes

- Physical datacenter presence + AWS EKS hybrid — both on-prem SAN/NAS and cloud block/object are in play
- Splunk SmartStore relevance: hot/warm on fast NAS or EBS, cold on S3-compatible object storage
- EKS stateful workloads: EBS (RWO default), EFS (RWX for shared), S3 via Mountpoint for large datasets
- Compliance/retention: telecom regulatory requirements drive long-term object storage architecture (CDRs, lawful intercept data)
- Scale: enterprise telco generates petabyte-class log volumes — object storage lifecycle policies are not optional
