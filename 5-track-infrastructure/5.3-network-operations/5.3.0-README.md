# Network Operations — SRE to Director Framework

## Why This Framework

Most software-origin SREs can debug a service mesh, tune a Kubernetes scheduler, or instrument a distributed trace — but they cannot explain why BGP route flaps cause application latency spikes, why DNS TTL is a business risk decision, or how peering economics work at carrier scale. That gap is exactly where Director-level interviews separate candidates. Network operations knowledge is the differentiator that elevates a strong SRE from "can run the platform" to "can architect and govern the infrastructure it depends on."

## Your Starting Advantage

T-Mobile is not a typical employer for this learning path — it is an accelerant. You have operated on the carrier side of BGP: you understand what it means for T-Mobile to have upstream transit relationships, peering agreements with content providers, and carrier-grade segmentation between signaling, management, and data planes. Most Director candidates at enterprise tech companies have only seen networks from the enterprise-customer side. You have seen both. That carrier fluency — MPLS backbone, SS7 signaling plane, multi-region HA with real 99.999% uptime requirements — maps directly to what enterprise Director and VP roles require and expect. You are not learning network operations from scratch. You are translating deep operational intuition into the vocabulary and decision frames that enterprise interviewers are looking for.

## Director Interview Readiness Checklist

Work through this list before any Director or VP of Platform Engineering interview. Each item maps to a chapter and represents a question style that appears in leadership-loop interviews.

### Network Architecture Decisions
- [ ] Can explain BGP failure modes and how they cascade to application incidents (ch02)
- [ ] Can describe multi-ISP architecture tradeoffs — active-active vs. active-passive, physical path diversity (ch08)
- [ ] Can explain when to use Direct Connect vs. public internet — cost, latency, compliance, failure mode tradeoffs (ch06)
- [ ] Can make the peering vs. transit economic argument — volume threshold, ASN requirements, IXP mechanics, payback timeline (ch08)
- [ ] Can describe how anycast DNS enables geographic load distribution and what happens when a PoP fails (ch03, ch05)

### Resilience and Reliability
- [ ] Can describe CDN failure modes and multi-CDN strategy — when to split traffic across two CDNs and how (ch04)
- [ ] Can design load balancing health checks that don't create failure cascades — threshold, interval, grace period tradeoffs (ch05)
- [ ] Can articulate DNS TTL strategy as a business risk decision — what the TTL means for failover time and who owns that decision (ch03, ch08)
- [ ] Can describe the blast radius question — what to ask in every network architecture review (ch08)

### Security and Segmentation
- [ ] Can describe DDoS response layers — edge absorption, upstream scrubbing, WAF — and when each layer applies (ch07)
- [ ] Can explain zero-trust vs. perimeter security and what microsegmentation means in a Kubernetes context (ch07)
- [ ] Can explain firewall rule sprawl, why it happens, and what organizational process prevents it (ch07)
- [ ] Can describe how Splunk is used for network security monitoring — VPC flow logs, DNS NXDOMAIN analysis, top talker detection (ch07)

### Vendor and Commercial Decisions
- [ ] Can describe what SLA terms to negotiate with an ISP — availability definition, credit provisions, MTTR commitment, escalation path (ch08)
- [ ] Can make the build vs. buy vs. hybrid decision for network capacity using the volume/control/compliance/team/capital matrix (ch08)
- [ ] Can explain why physical path diversity matters and how to verify it in ISP contracts (ch08)

### T-Mobile Context Available for STAR Stories
- [ ] BGP from the carrier side — T-Mobile's upstream transit and peering relationships (ch01, ch02, ch08)
- [ ] 25M msg/day notification platform — network reliability requirements that flow from that SLA (ch01, ch05)
- [ ] Three-plane carrier segmentation — signaling, management, and data plane separation (ch07)
- [ ] Splunk for network log analysis at scale — VPC flow logs, firewall logs, DNS anomaly detection (ch07)

## Prerequisites

Before starting ch01, you should be able to:
- Explain what TCP vs. UDP is and when each is used (operational, not academic)
- Explain what an IP address and subnet mask represent
- Navigate a Splunk search interface and write basic stats queries
- Describe what a load balancer does and why applications use one
- Explain what DNS does at a 30-second level — "translates names to IPs"

You do not need to know BGP syntax, firewall CLI commands, or network hardware specifics before starting. Those come in context within each chapter.
