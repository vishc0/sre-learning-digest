# Hypervisor Comparison Matrix

Quick-reference for platform selection and interview conversations. Five columns: VMware vSphere, Proxmox VE, Nutanix AOS (with AHV), KVM (bare), Microsoft Hyper-V.

## Core Feature Matrix

| Feature | VMware vSphere | Proxmox VE | Nutanix AOS + AHV | KVM (bare) | Microsoft Hyper-V |
|---------|---------------|-----------|------------------|-----------|------------------|
| **License model** | Subscription (Broadcom, 2024+). No more perpetual. | Open-source (free). Commercial support via Proxmox GmbH. | Subscription. Per-node or per-core pricing. | Open-source (GPL). Part of Linux kernel. | Included with Windows Server. Separate licensing for SCVMM management. |
| **Cost (ballpark, per node/yr)** | $80-250/core/yr (VVF/VCF). 16-core minimum per CPU. | $0 (community). ~$1,200/yr/node for enterprise support. | $8,000-15,000/node/yr (hardware + software bundle). Varies significantly by volume. | $0 software. Ops cost is the cost. | $0 with Windows Server Datacenter license. SCVMM ~$700/server/yr. |
| **Live migration** | vMotion — mature, well-integrated. Zero-downtime. CPU compatibility: Enhanced vMotion Compatibility (EVC) handles CPU generation differences. | Live migration via PVE API. Works well within cluster. Requires shared storage or CEPH. | Nutanix AHV live migration — built into Prism. Works across nodes within cluster. | virsh migrate — functional but requires manual coordination. Not for the faint-hearted. | Hyper-V Live Migration — mature. Cluster Shared Volumes (CSV) for shared storage. |
| **HA clustering** | vSphere HA — automatic VM restart on host failure. Industry standard. | HA via Proxmox HA Manager. Uses Corosync for quorum. Functional but requires configuration expertise. | Native HA in Nutanix AOS. VMs restart automatically on node failure. Managed by Prism. | No built-in HA. Requires Pacemaker/Corosync for HA orchestration. Significant ops overhead. | Failover Clustering — Windows-native. Mature for Windows VMs. Less natural for Linux VMs. |
| **Storage virtualization** | vSAN — mature, enterprise-grade. All-flash or hybrid. Integrated with vSphere. Dedup, compression, encryption. | Ceph integration — open-source distributed storage. Or ZFS local storage with replication. Powerful but complex to operate. | Nutanix DSF (Distributed Storage Fabric) — tightly integrated with AHV. Excellent performance. Dedup, compression, tiering. | Ceph or GlusterFS — separate projects. Integration is manual. | Storage Spaces Direct (S2D) — Windows-native HCI storage. Equivalent to vSAN but for Hyper-V. Mature for Windows-heavy environments. |
| **Network virtualization** | NSX — full SDN. Routing, firewall, LB, NAT. Microsegmentation. Industry-leading for VM environments. | OVS (Open vSwitch) — integrated. VXLAN overlay support. No built-in microsegmentation. | Nutanix Flow — SDN layer for AHV. Microsegmentation policies. Less mature than NSX but functional. | OVS or Linux bridges. No built-in SDN management. Manual configuration required. | Hyper-V Virtual Switch — basic. Software-defined networking via SCVMM + HNV (Hyper-V Network Virtualization). |
| **GUI management** | vCenter — polished, comprehensive. Web-based HTML5 client. Excellent inventory, performance, and alarm views. | Proxmox web UI — solid for an open-source project. All features accessible. Less polished than vCenter. | Prism Element + Prism Central — modern, clean. Prism Central manages multiple clusters from one UI. | No GUI by default. Virt-manager for per-host GUI. Cockpit for basic management. Enterprise users add oVirt. | Hyper-V Manager (basic) or System Center VMM (SCVMM) for enterprise management. SCVMM UI is dated. |
| **Enterprise support** | Broadcom Global Support — enterprise SLAs available. Quality perception has declined post-acquisition per customer reports. | Proxmox GmbH commercial support. Limited team size. For large-scale environments, relies on community + self-service. | Nutanix Support — generally rated highly. Dedicated support tiers. Strong customer satisfaction scores in analyst surveys. | Redhat (for RHEL-KVM) or community. No "KVM vendor" for bare deployments. | Microsoft CSS — mature, large organization. SLA-backed for commercial customers. |
| **Kubernetes integration** | vSphere with Tanzu — native K8s supervisor. TKG for standalone clusters. Most mature VMware-native K8s path. | No native K8s orchestration. Can run K8s VMs like any other VM. Minimal integration. | Nutanix Kubernetes Engine (NKE) — managed K8s on Nutanix. Integrates with Nutanix storage and networking. | Use kubeadm on KVM VMs. No hypervisor-level integration. Standard approach for OpenStack + K8s. | AKS on Azure Stack HCI — managed Kubernetes on Hyper-V-based HCI. Strong for Microsoft-shop environments. |
| **Broadcom impact** | Direct — this IS the Broadcom product. Pricing changes are ongoing. | None — independent open-source project. Beneficiary of VMware customer displacement. Proxmox GmbH saw record growth in 2024. | Indirect positive impact — Nutanix is the most common VMware migration target. Nutanix grew significantly from VMware displacement. | None — KVM is kernel code, not a vendor product. | None — Microsoft has different motivations. Hyper-V not Broadcom-affected. |

## Decision Guide: When to Choose Each

| If you need... | Best choice | Why |
|---------------|-------------|-----|
| Maximum enterprise feature set, existing VMware investment | VMware vSphere | Depth of features, vMotion/HA/DRS maturity, NSX |
| On-prem HCI replacement for VMware, avoid Broadcom | Nutanix AOS + AHV | Closest functional equivalent, best migration tooling, enterprise support |
| Budget-constrained, open-source OK, Linux-skilled team | Proxmox VE | Zero license cost, active community, improving enterprise support |
| Maximum control, cloud-like IaaS on-prem, large ops team | KVM + OpenStack | Full stack control, cloud-native API patterns, open source |
| Microsoft-centric datacenter, Windows VMs dominant | Hyper-V | Included in Windows Server, native Active Directory integration, AKS-HCI path |
| Cloud migration endpoint | AWS EC2 / Azure VM | Managed infrastructure, global reach, cloud-native services |

## Functional Equivalence Map

For Vishweshwar's mental model — mapping VMware concepts to alternatives:

| VMware Concept | Nutanix Equivalent | Proxmox Equivalent | KVM Equivalent | Kubernetes Equivalent |
|---------------|-------------------|-------------------|---------------|----------------------|
| vCenter | Prism Central | Proxmox API + Web UI | oVirt / Cockpit | Kubernetes API server |
| ESXi | AHV hypervisor | KVM + QEMU (on Proxmox) | KVM + QEMU | kubelet + containerd |
| vSAN | Nutanix DSF | Ceph (via Proxmox) | Ceph (manual) | Rook-Ceph + PVC |
| NSX | Nutanix Flow | OVS + VXLAN | OVS + Neutron | Cilium / Calico |
| vMotion | AHV live migration | Proxmox live migration | virsh migrate | Pod rescheduling |
| HA | Nutanix HA | Proxmox HA Manager | Pacemaker | ReplicaSet controller |
| DRS | Nutanix ADS (Acropolis Dynamic Scheduler) | None (manual) | None (manual) | kube-scheduler |
| SRM | Nutanix DR (Xi Leap) | Manual runbook | Manual runbook | Velero + GitOps |
