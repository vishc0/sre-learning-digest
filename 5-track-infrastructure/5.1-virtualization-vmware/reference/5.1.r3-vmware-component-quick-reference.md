# VMware / Broadcom Product Quick Reference

Current as of mid-2024. Broadcom renamed/rebranded several products post-acquisition. This reference uses both old and new names.

## Core Platform Components

| Product (Old Name) | Current Broadcom Name | What It Does | Where It Runs | Key Fact |
|-------------------|-----------------------|-------------|--------------|---------|
| **ESXi** | VMware ESXi (unchanged) | Type 1 bare-metal hypervisor. The foundation that runs VMs directly on physical hardware. | Physical server hardware | The only product that actually runs on bare metal. Everything else is a VM or software layer. |
| **vCenter Server** | VMware vCenter (unchanged) | Centralized management plane for ESXi hosts and VMs. GUI, API, automation, RBAC, HA/DRS orchestration. | Runs as VCSA (Linux appliance VM) on an ESXi host | vCenter itself must be highly available — if it goes down, VMs keep running but you lose management capability. |
| **vSphere** | VMware vSphere (unchanged) | Not a single product — it's the brand name for the combined ESXi + vCenter platform. | N/A — it's a platform label | When someone says "vSphere environment" they mean ESXi hosts managed by vCenter. |
| **vSphere with Tanzu** | vSphere with Tanzu (VMware by Broadcom) | Kubernetes supervisor cluster running directly on vSphere. Enables running Kubernetes workloads alongside VMs. | Supervisor runs on vSphere cluster | Not the same as TKG — this is native K8s integration into vSphere itself. |

## Storage Products

| Product (Old Name) | Current Broadcom Name | What It Does | Key Fact |
|-------------------|-----------------------|-------------|---------|
| **vSAN** | VMware vSAN | Software-defined storage that pools local ESXi host disks into a distributed shared datastore. Replaces traditional SAN/NAS for VM storage. | Requires minimum 3 ESXi hosts. Licensing now bundled in VVF and VCF. |
| **vSAN Max** | VMware vSAN Max | Disaggregated vSAN — dedicated storage nodes separate from compute nodes. Enables larger storage scale independent of compute. | For large environments where compute and storage scale at different rates. |
| **vSphere Replication** | vSphere Replication (unchanged) | VM-level replication from source site to recovery site. RPO as low as 5 minutes. | Not to be confused with vSAN mirroring — vSphere Replication works across sites for DR. |
| **Site Recovery Manager (SRM)** | VMware Live Recovery (rebranded) | DR orchestration. Runs scripted, ordered failover of VM groups across sites using vSphere Replication or array-based replication. | SRM is orchestration only — it does not replicate data, it automates the failover. |

## Networking Products

| Product (Old Name) | Current Broadcom Name | What It Does | Key Fact |
|-------------------|-----------------------|-------------|---------|
| **NSX-T** | VMware NSX | Software-defined networking overlay. Provides routing, switching, firewall, LB, NAT in software on ESXi hosts. Replaced the older NSX-V product. | NSX-T (the current version) is NSX for vSphere AND for non-VMware environments (bare metal, containers). |
| **NSX Advanced Load Balancer (Avi)** | VMware NSX ALB (Avi) | Full-featured application delivery controller (ADC). Software load balancer, WAF, analytics. Replaces hardware LB appliances. | Broadcom acquired Avi Networks in 2019. Now the preferred LB for NSX environments. |
| **HCX** | VMware HCX | Hybrid Cloud Extension — WAN-optimized VM migration between on-prem and cloud (or site to site). Live migration over WAN, no L2 stretch required. | Used for large-scale migration projects. Commonly used in VMware on AWS deployments to migrate from on-prem vSphere. |
| **vCloud Director** | VMware Cloud Director | Multi-tenancy management plane for service providers. Lets hosting providers offer VMware-based cloud services to customers. | Primarily a service provider product, not typical enterprise IT. |

## Management and Operations Products

| Product (Old Name) | Current Broadcom Name | What It Does | Key Fact |
|-------------------|-----------------------|-------------|---------|
| **vRealize Operations** | VMware Aria Operations | Infrastructure monitoring, capacity planning, and workload optimization for VMware environments. | "Aria" is Broadcom's rebrand of the entire vRealize suite. |
| **vRealize Log Insight** | VMware Aria Operations for Logs | Log aggregation and analysis for VMware and general infrastructure. | Like Splunk but VMware-specific and less capable for general log analytics. |
| **vRealize Automation** | VMware Aria Automation | Self-service cloud management platform. Provisions VMs, cloud resources, and Kubernetes clusters via catalog-based interface. | Think of it as ServiceNow for infrastructure provisioning — catalog-driven. |
| **vRealize Network Insight** | VMware Aria Operations for Networks | Network visibility, traffic analysis, and security planning for NSX environments. | Used for microsegmentation planning — shows actual VM-to-VM traffic flows to help design firewall rules. |
| **vRealize Suite Lifecycle Manager** | VMware Aria Suite Lifecycle | Installs, upgrades, and manages the Aria (vRealize) product stack. | Meta-management: manages the managers. |

## Kubernetes and Modern App Products

| Product (Old Name) | Current Broadcom Name | What It Does | Key Fact |
|-------------------|-----------------------|-------------|---------|
| **Tanzu Kubernetes Grid (TKG)** | VMware Tanzu Kubernetes Grid | Deploys and manages upstream Kubernetes clusters on vSphere, AWS, Azure, or bare metal. | TKG is VMware's distribution of Kubernetes — like EKS but multi-cloud and on-prem. |
| **Tanzu Application Platform (TAP)** | VMware Tanzu Platform | Developer-focused platform for building and running apps on Kubernetes. Supply chain automation, service bindings, developer portal. | Competes with Red Hat OpenShift. Very opinionated developer experience on top of K8s. |
| **Tanzu Mission Control** | VMware Tanzu Mission Control | Centralized management for multiple Kubernetes clusters across environments. | Like EKS Control Tower — manage many clusters from one plane. |

## Broadcom Licensing Bundles (New as of 2024)

| Bundle Name | What It Includes | Best Fit | Roughly Excludes |
|------------|-----------------|----------|-----------------|
| **VMware vSphere Essentials Plus** | ESXi + vCenter, max 3 hosts / 96 cores | Small business | NSX, vSAN, advanced features |
| **VMware vSphere Foundation (VVF)** | ESXi + vCenter + vSAN + basic Aria Operations | Mid-market / enterprise without NSX | NSX, advanced Aria |
| **VMware Cloud Foundation (VCF)** | ESXi + vCenter + vSAN + NSX + full Aria Suite | Full software-defined datacenter | Nothing significant — this is the top bundle |
| **VMware Tanzu Platform** | Kubernetes + developer platform on top of vSphere | Cloud-native/developer-focused organizations | Traditional VM management features |
