## AGENDA

### ITEM 1: Backlog Health Snapshot (5 minutes)

| | |
|---|---|
| **Owner** | Problem Manager |
| **Time** | 5 minutes |
| **Decision Required** | No |
| **Input** | Problem Backlog Dashboard (pre-read) |
| **Expected Output** | Group is aligned on backlog state; anomalies flagged for discussion |

**Discussion points**:
- Current backlog size by priority (P1 / P2 / P3 / P4) vs. last week
- Average age by priority vs. SLA targets
- Any priority changes since last meeting
- Problems approaching SLA breach (flag for escalation in Item 2 if needed)

**Metrics to review** _(pulled from dashboard, not discussed one-by-one — only outliers flagged)_:
- Problem Backlog Size: current vs. target
- Backlog Age (average by priority): current vs. SLA
- Fix Delivery Rate (last 30 days): current vs. target
- Recurrence Rate (last 90 days): current vs. target

---

### ITEM 2: P1 / P2 Status Review (15 minutes)

| | |
|---|---|
| **Owner** | Problem Owner for each active P1/P2 |
| **Time** | 15 minutes total — max 5 minutes per problem |
| **Decision Required** | Sometimes — see below |
| **Input** | P1/P2 status updates (pre-read) |
| **Expected Output** | Current status confirmed; blockers surfaced; escalation decisions made |

**Format for each problem** (Problem Owner presents, 3 minutes max):
1. Current status (one sentence)
2. Progress since last review (what changed)
3. Next milestone and date
4. Blockers — if any (only blockers that require group action; operational updates belong in the problem record, not here)

**Decisions this item may require**:
- Escalation to Service Owner or executive if fix delivery is at risk
- Priority adjustment if new evidence changes impact assessment
- Resource reallocation if problem is blocked by team capacity

**Time discipline**: If no update since last week and no blockers, state that in 30 seconds and move on. This is not a status theater exercise.

---

### ITEM 3: RCA Reviews and Approvals (10 minutes)

| | |
|---|---|
| **Owner** | Problem Manager / RCA Facilitator |
| **Time** | 10 minutes — max 5 minutes per RCA |
| **Decision Required** | Yes — approval or rejection of RCA reports |
| **Input** | RCA reports in Final status (pre-read) |
| **Expected Output** | RCA approved and moved to Approved status, OR rejection with specific feedback |

**For each RCA awaiting approval**:
1. Facilitator presents: what happened, root cause headline, corrective actions (2 minutes max — attendees have read the report)
2. Group: challenge the root cause — is it structural condition or symptom? (1 minute)
3. Group: verify corrective actions — is there at least one Prevention action addressing the structural condition? Are all owners named individuals? Are verification methods specific? (1 minute)
4. Decision: Approve / Reject with specific feedback / Table for next week with specific questions answered

**RCA rejection criteria** (any of these = reject):
- Root cause statement describes a symptom, not a structural condition
- No Prevention-type corrective action present
- Corrective actions lack named individual owners
- Evidence gaps are not documented
- Contributing factors list has no Process or Organizational items

---

### ITEM 4: New Problems Triage (10 minutes)

| | |
|---|---|
| **Owner** | Problem Manager leads; Problem Owners for new records present |
| **Time** | 10 minutes — max 3 minutes per new problem |
| **Decision Required** | Yes — priority assignment, ownership assignment |
| **Input** | New problems opened since last review (pre-read) |
| **Expected Output** | Each new problem has confirmed priority, owner, and initial RCA/investigation timeline |

**For each new problem**:
1. Problem Owner presents: short description, affected CIs, related incidents, initial priority score (2 minutes)
2. Group challenge: is the priority score correct? Have all dimensions been considered?
3. Decisions: confirm or adjust priority; confirm owner; set initial investigation milestone date

**Priority scoring must be presented** — "I think it's a P2" without a score is not acceptable. If score has not been calculated, problem is not ready for triage — defer to next week with a requirement to complete scoring first.

---

### ITEM 5: Overdue Action Review (5 minutes)

| | |
|---|---|
| **Owner** | Problem Manager |
| **Time** | 5 minutes |
| **Decision Required** | Sometimes — re-commitment or escalation |
| **Input** | Overdue actions list (pre-read) |
| **Expected Output** | Each overdue action has new committed date or escalation decision |

**For each overdue action**:
1. Action owner states: why overdue and new committed date (30 seconds max)
2. If overdue for second consecutive week: Problem Manager escalates to Service Owner — document in Decision Log below

**No debate in the meeting** about why something is late. That is a 1:1 conversation. The group only needs to hear the new date or the escalation decision.

---

### ITEM 6: Process / Tooling / Escalations (5 minutes)

| | |
|---|---|
| **Owner** | Chair |
| **Time** | 5 minutes |
| **Decision Required** | Sometimes |
| **Input** | Anything raised by attendees not covered by Items 1–5 |
| **Expected Output** | Items resolved, parked, or assigned |

Use this slot for:
- Tooling or process issues that affect the problem management program
- Cross-team coordination needs identified during the week
- Upcoming capacity constraints affecting fix delivery
- Requests to suspend or archive KEDB entries

Items that need more than 2 minutes get parked — owner assigned, follow-up offline.
