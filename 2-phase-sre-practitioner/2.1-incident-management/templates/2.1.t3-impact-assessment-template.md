# Impact Assessment Template
**Purpose**: Structured scoring during active incidents. Update Sections A and B every 30 minutes. Section C is finalized at mitigation for PIR input.

**When to use**: Start filling within 10 minutes of SEV1/SEV2 declaration. For SEV3/SEV4, complete at declaration close.

---

```
Incident ID  : [INC-XXXXX]
Service      : [primary affected service]
Assessment   : [initial | 30-min update | 60-min update | final]
Completed by : [name]

```

Journey Criticality Key:
- **P0**: Revenue-generating or safety-critical — every minute of impact is material

- **P2**: Supporting functionality — degrades experience but workaround exists

### A4. BRI (Business Risk Index) Calculation

**Formula**: `BRI = (affected_users / total_users) × journey_criticality × tier_weight_of_failing_service`

| Variable                     | Value              | Source / Notes                                      |
|------------------------------|--------------------|-----------------------------------------------------|
| Affected users / Total       | [X%]               | A1 above                                            |
| Journey criticality          | [1.0 / 0.7 / 0.5 / 0.3] | 1.0=checkout/auth/payment, 0.7=search/account, 0.5=recommendations, 0.3=reviews/analytics |
| Tier weight (failing service)| [1.0 / 0.8 / 0.5 / 0.3] | P0=1.0, P1=0.8, P2=0.5, P3=0.3                |
| **BRI Score**                | **[calculated]**   |                                                     |

**BRI → SEV Mapping** (canonical — from SRECapstone framework):

| BRI Score  | SEV Level | Action Required                                       |
|------------|-----------|-------------------------------------------------------|
| > 0.70     | SEV1      | Immediate all-hands; executive notification            |
| 0.30–0.70  | SEV2      | On-call team + engineering management                 |
| 0.10–0.30  | SEV3      | On-call team; monitor closely                         |
| < 0.10     | SEV4      | Engineering queue; next business day OK               |

**Multi-dimension rule**: Always classify by the highest dimension across BRI, revenue, users affected, and regulatory exposure. BRI at SEV3 does not prevent SEV2 if revenue exposure is in the SEV2 range ($1K–$10K/min).

## SECTION A — CUSTOMER IMPACT

### A1. Affected Users

| Metric                        | Value      | Confidence | Notes                               |
|-------------------------------|------------|------------|-------------------------------------|

| Affected users (%)            | ~68%       | M          | 12,200 / 18,000                     |

- [x] Consumer (mobile): YES — same backend
- [ ] Enterprise / B2B: NO — B2B uses separate payment rail

**BRI 0.544 → SEV2** (validated — initial SEV1 declaration confirmed appropriate given velocity and trajectory)
