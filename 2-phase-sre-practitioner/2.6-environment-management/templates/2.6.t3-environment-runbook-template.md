## Restart Procedures

### Restart a Single Service

```bash
# Option 1: Rolling restart (preferred — zero downtime)
kubectl rollout restart deployment/[SERVICE_NAME] -n [ENVIRONMENT_NAME]

# Verify restart completes
kubectl rollout status deployment/[SERVICE_NAME] -n [ENVIRONMENT_NAME]
# Expected: "deployment "[SERVICE_NAME]" successfully rolled out"

# Option 2: Delete pod (forces immediate restart)
kubectl delete pod -l app=[SERVICE_NAME] -n [ENVIRONMENT_NAME]
```

**Expected recovery time**: 60–120 seconds for most services.

### Restart All Services

```bash
# Rolling restart of all deployments in the namespace
kubectl rollout restart deployment --all -n [ENVIRONMENT_NAME]

# Monitor restart progress
kubectl get pods -n [ENVIRONMENT_NAME] -w
# Expected: All pods in Running state within 5 minutes
```

### Full Environment Rebuild (Last Resort)

Use when service restarts do not resolve the issue and configuration appears corrupted.

```bash
# Step 1: Capture current configuration for reference
kubectl get all -n [ENVIRONMENT_NAME] -o yaml > /tmp/pre-rebuild-state-$(date +%Y%m%d-%H%M%S).yaml

# Step 2: Helm uninstall all releases
for RELEASE in $(helm list -n [ENVIRONMENT_NAME] -q); do
  helm uninstall $RELEASE -n [ENVIRONMENT_NAME]
done

# Step 3: Wait for namespace to clear
kubectl get pods -n [ENVIRONMENT_NAME]
# Expected: No pods remaining

# Step 4: Redeploy from Helm charts
cd [ENVIRONMENT_CONFIG_REPO]
./scripts/deploy-environment.sh --environment [ENVIRONMENT_NAME] --tier [TIER]

# Step 5: Validate health
./scripts/env-health-check.sh --environment [ENVIRONMENT_NAME]
```

## Monitoring and Alerting

### Key Dashboards

| Dashboard | What to Watch | Alert Threshold |
|-----------|-------------|----------------|
| Pod health | All pods in Running state | Any pod in CrashLoopBackOff → P2 |
| Service latency | p99 endpoint latency | > [N]ms → investigate |
| Error rate | HTTP 5xx rate | > 1% → investigate |
| Database connections | Connection pool utilization | > 80% → P2 |
| Memory utilization | Pod memory vs. limit | > 85% → P2 |

### Alert Recipients

| Alert | Recipient |
|-------|----------|
| Pod crashloop | Primary owner + platform SRE on-call |
| Service unavailable | Primary owner + all teams using this environment |
