# Configuration Parity Checklist

**Purpose**: Layer-by-layer checklist for verifying that a non-production environment matches production configuration. Run before any environment is used for its first test cycle, and after any significant infrastructure or middleware change in production.

**Audience**: Environment owners, platform SRE, release managers (Staging validation).

**Usage**: Complete one checklist per environment per validation cycle. Retain completed checklists for compliance evidence.
