## CLUSTER 1: STRUCTURAL

### CD — Chain Depth
```
CD = count(sequential_synchronous_hops_in_journey)
```
**Question answered:** How many services can fail this journey?
**Rule:** CD is a budget — enforce a maximum per journey tier.

---

### Required SLO
```
Required_SLO = customer_SLA ^ (1/CD)
```
**Question answered:** What per-service SLO do I need to hit the customer SLA?
**Key threshold:** Each additional hop raises the required per-service SLO.
**Example:** SLA=99.95%, CD=5 → 99.95%^(1/5) = 99.990% per service required.

---

### SCS — SLO Coherence Score
```
SCS = (product of all declared_SLOs) / customer_SLA
    = declared_SLO_1 × declared_SLO_2 × ... × declared_SLO_n / SLA
```
**Question answered:** Do our declared SLOs actually cover the customer SLA?
**Key threshold:** Target 0.95 – 1.10. Below 0.95 = SLA breach risk. Above 1.10 = over-engineered.

---

### LBH — Latency Budget per Hop
```
LBH = (end_to_end_p99_target - infra_overhead) / CD
```
**Question answered:** How fast must each service respond at p99?
**Key threshold:** If LBH < 200ms, any cache miss will breach. Flag for architecture review.

---

## CLUSTER 2: BLAST RADIUS

### DG — Dependency Gravity
```
DG = Σ[tier_weight × proximity_discount]
     for all deps and callers within 2 hops
```
**Question answered:** How much of the system does this service touch, weighted by criticality?
**Key threshold:** DG > 10 = high gravity. Consider as P0 regardless of declared tier.

---

### BRI — Blast Radius Index
```
BRI = (affected_users / total_users) × journey_criticality × tier_weight_of_failing_service
```
**Question answered:** How severe is this incident right now?
**Key threshold:** See BRI Severity table above.

---

### CC — Cascade Coupling
```
CC = Σ[P(dep_fails | X_fails) × tier_weight(dep)]
     P = 0.9 (no CB) | 0.5 (partial CB) | 0 (full CB)
```
**Question answered:** If this service fails, how likely is cascade and how bad?
**Key threshold:** CC > 5 = requires circuit breakers. CC > 10 = JRCS will be negative.

---

### FLI — Failure Locality Index
```
FLI = contained_failure_events / total_failure_events
```
**Question answered:** Is the system able to contain failures at their source?
**Key threshold:** Target > 0.90. Below 0.70 = circuit breakers urgently needed.

---

### DSA — Dependency Surface Area
```
DSA = Σ[1.0 × tw_direct] + Σ[0.5 × tw_transitive]
```
**Question answered:** How exposed is this service through both direct and transitive paths?
**Key threshold:** DSA > 4 = high surface area. Override tier assignment for change governance.

---

## CLUSTER 3: HEALTH SIGNALS

### EBV — Error Budget Velocity
```
EBV = traffic_normalized_error_rate / baseline_rate
```
**Question answered:** How fast am I burning error budget vs. plan?
**Key threshold:** EBV > 1.0 = burning above plan. EBV > 2.0 = freeze non-critical changes.

---

### EDFD — Error Budget Exhaustion Date
```
EDFD = today + (remaining_budget / (EBV × baseline_rate))
```
**Question answered:** When will the error budget run out at current burn rate?
**Key threshold:** EDFD < end_of_month = change freeze required.

---

### APR — Actionable Page Ratio
```
APR = actionable_alerts / total_alerts
```
**Question answered:** What fraction of alerts require SRE action?
**Key threshold:** Target > 0.80. Below 0.50 = alert redesign required immediately.

---

### MTBI / Achievable SLO
```
MTBI = period / incident_count

achievable_SLO = MTBI / (MTBI + MTTR)
```
**Question answered:** What SLO can this service actually achieve given its operational data?
**Key threshold:** If achievable_SLO < declared_SLO = SLO theater. Fix or renegotiate.

---

### OCR — Observability Coverage Ratio
```
OCR = Σ(coverage_fraction × tier_weight) / Σ(tier_weight)
      coverage = fraction of 4 signals present (metrics, logs, traces, alerts)
```
**Question answered:** What fraction of the weighted service surface can I observe?
**Key threshold:** Target > 0.70. Below 0.50 = MRI amplification — all changes become riskier.

---

## CLUSTER 4: EFFICIENCY

### OL — Operational Load
```
OL = Δ(budget_percentage_points_per_week) / SRE_hours_per_week
```
**Question answered:** How many budget points does the team consume per SRE-hour of work?

---

### RCR — Recurring Cost Ratio
```
RCR = (response_hours × frequency_per_year) / fix_hours
```
**Question answered:** Should I automate this toil now or defer?
**Key threshold:** RCR > 0.50 = fix. RCR < 0.10 = defer. Between = judgment call.

---

### TAF — Toil Amplification Factor
```
TAF = toil_growth_rate / traffic_growth_rate
```
**Question answered:** Is toil growing faster than the system it serves?
**Key threshold:** Target < 1.0. TAF > 1.0 = toil will eventually consume the team. TAF > 1.5 = urgent automation investment required.

---

### RIS — Reliability Investment Score
```
RIS = (RCR × OL) / MRI
```
**Question answered:** Which reliability investment delivers the best return per SRE-hour?
**Use:** Rank investment options. Higher RIS = higher priority. Supplement with strategic judgment — low-RIS prerequisites still matter.

---

## CLUSTER 5: CHANGE GOVERNANCE

### SCV — Service Change Velocity
```
SCV = deploy_frequency_per_week × mean_CSAI × failure_rate
      mean_CSAI = (Σ CSAI values) / deploy_count
```
**Question answered:** How much change risk is accumulating in this service right now?
**Key threshold:** < 0.50 stable | 0.50–1.0 moderate | > 1.0 high-risk.

---

### MRI — Marginal Risk Index
```
MRI = DG × CD × (1 - FLI) × (2 - OCR_weighted)
```
**Question answered:** How dangerous is it to make any change to this service right now?
**Key threshold:** See MRI Approval Tiers table above.
**Note:** MRI measures system state, not change content. Fix FLI and OCR to lower MRI.

---

### CSD — Change Surface Depth
```
CSD = Σ[tier_weight × depth_coefficient]
      for each component changed, across all services touched
```
**Question answered:** How deep and broad is this change across the system?
**Key threshold:** < 5 small | 5–10 medium (expand-contract pattern required) | > 10 large (CAB escalation).

---

### DSS — Deployment Safety Score
```
DSS = (1 - SCV_norm) × (1 - MRI/10) × APR × budget_remaining_fraction
      SCV_norm = min(SCV/5, 1.0)
```
**Question answered:** Is it safe to deploy right now?
**Key threshold:** See DSS Monitoring Tiers table above.

---

### EBR — Error Budget Reserve
```
EBR = min(MRI × 0.05, 0.80)
      [fraction of monthly budget this change must reserve]
```
**Question answered:** Does enough error budget remain to cover this change's risk?
**Key threshold:** EBR must be ≤ budget_remaining_fraction. Otherwise: budget block.

---

### Window Sizing (RV)
```
deployment_window ≥ 3 × rollback_time (RV)
```
**Question answered:** Is the deployment window long enough to detect and roll back?
**Key threshold:** 3×RV is minimum. Recommended: 3.5×RV to include decision time.

---

## CLUSTER 6: LIFECYCLE

### GEI — Growth-to-Entropy Index
```
GEI = (new_dependencies × avg_tier_weight) / (test_coverage × canary_hours)
```
**Question answered:** How much complexity risk does this new integration add?
**Key threshold:** < 1.0 low risk | 1.0–3.0 requires extended canary | > 3.0 block.

---

### RST — Rollout Signal Threshold
```
RST = rollout_percentage where (feature_error_rate - baseline_error_rate) / σ_baseline ≥ 2
```
**Question answered:** At what rollout percentage should I stop and investigate feature impact?
**Key threshold:** z-score ≥ 2 = statistically significant. Stop rollout, investigate.

---

### RDR — Regression Detection Rate
```
RDR = new_failure_mode_rate / old_failure_mode_rate
```
**Question answered:** After a change, did regression rate decrease as expected?
**Key threshold:** Target < 0.10 (90% reduction). > 0.50 = regression not contained.

---

### SCI — Silent Caller Index
```
SCI = (observed_rps - declared_rps) / total_observed_rps
```
**Question answered:** Are there undocumented callers before I decommission?
**Key threshold:** Must = 0.0 exactly. Any SCI > 0 = BLOCKED from decommissioning.
**Observation period:** 30 consecutive days of SCI=0 required before proceeding.

---

## CLUSTER 7: COMPOSITE SCORES

### SRMI — System Reliability Maturity Index
```
SRMI = (FLI × OCR × APR/0.85) / (TAF × CC_max/10)
```
**Question answered:** How mature is this system's overall reliability posture?
**Key threshold:** Score approaching 1.0 = mature. Below 0.30 = high-risk for traffic events.

---

### JRCS — Journey Reliability Coherence Score
```
JRCS = SCS × FLI × OCR_path × (1 - CC_max/10)
```
**Question answered:** Is this customer journey systemically safe end-to-end?
**Key threshold:** > 0 = safe. Negative = CC_max > 10, cascade risk exceeds safety margin.
**Warning:** If CC_max > 10, JRCS will always be negative regardless of other metrics.

## FORMULA DEPENDENCY MAP

```
System Health Inputs:
  FLI, OCR, APR, TAF, CC_max
       │
       ├──► MRI (change governance gate)
       │        └──► EBR (budget gate)
       │        └──► DSS (deployment safety)
       │
       ├──► SRMI (overall maturity score)
       │
       └──► JRCS (journey safety score)

Service Structure:
  CD, SLA
    │
    ├──► Required_SLO (per-service target)
    ├──► LBH (per-hop latency target)
    └──► SCS (SLO coherence check)

Blast Radius:
  DG, BRI, CC, FLI, DSA
    │
    ├──► Severity declaration (BRI)
    ├──► Circuit breaker priority (CC)
    └──► Operational tier override (DG, DSA)
```
