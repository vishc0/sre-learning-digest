# Release Readiness Playbook

Release readiness is the systematic gate between "change approved" and "change in production." It is not a single sign-off — it is a structured verification that all conditions for a safe deployment are met before the change window opens.

---

## Pre-Release Gate (24 hours before change window)

**Technical readiness:**
- [ ] Artifact validated: container image, Helm chart, or IaC module is tagged, immutable, and stored in artifact registry
- [ ] Staging deployment confirmed: the exact artifact being deployed to production has been deployed and validated in staging within the past 72 hours
- [ ] Rollback artifact confirmed: the current production artifact (the one being replaced) is accessible for rollback — know the version, know how to deploy it
- [ ] Monitoring in place: deployment health metrics are defined and dashboards are ready. What metrics will you watch during deployment? What are the thresholds for rollback trigger?
- [ ] Feature flags configured: if the release uses feature flags, the flag state is confirmed. Who is in the initial cohort? What is the expansion plan?

**Process readiness:**
- [ ] RFC is CAB-approved with confirmed window
- [ ] Change window confirmed with on-call team lead
- [ ] Stakeholder notifications sent (24-hour pre-notice)
- [ ] Rollback owner is confirmed available during the change window
- [ ] Post-change monitoring owner is confirmed (who is watching for 2 hours after deployment?)

---

## Go/No-Go Decision (1 hour before change window)

Mandatory go/no-go call. Participants: change owner, on-call lead, Release Manager.

**Go criteria:**
- [ ] Artifact ready and accessible
- [ ] Rollback artifact ready and accessible
- [ ] Staging validation completed within past 72 hours
- [ ] Rollback owner confirmed available
- [ ] Monitoring dashboard open and showing baseline
- [ ] No active SEV1 or SEV2 incidents in the affected service group
- [ ] No other changes in the same window for the same services

**No-go criteria — any one condition = no-go:**
- Active SEV1/SEV2 incident
- Rollback owner unavailable
- Staging validation older than 72 hours
- Artifact not accessible
- Monitoring dashboard not available

**No-go outcome:** defer the change to next available window. Document the no-go reason in the change ticket. Notify CAB Chair and stakeholders.

---

## Implementation (during change window)

**Step 1:** Confirm all parties on change war room (Slack/Teams channel). Attendance: change owner, on-call lead, release manager, rollback owner.

**Step 2:** Announce change start in war room with timestamp.

**Step 3:** Execute implementation steps from RFC in order. Each step: announce to war room, execute, confirm outcome matches RFC expected output.

**Step 4:** At each defined verification point in the RFC, explicitly check go/no-go criteria before proceeding to next step. If criteria not met, call rollback.

**Step 5:** Announce change completion in war room with timestamp.

---

## Rollback Confirmation

After implementation and before closing the change window:
- [ ] Rollback owner confirms they are still available for 2-hour post-deployment observation window
- [ ] Rollback procedure is accessible — the RFC rollback steps are open, not just in someone's head
- [ ] Rollback trigger criteria are being actively monitored

---

## Post-Implementation (0–2 hours after deployment)

**Monitoring window:**
- P1/P2 changes: 2 hours minimum
- P3 changes: 1 hour minimum

**What to monitor:** error rates, latency percentiles, throughput, queue depth (for RabbitMQ/Kafka), and any service-specific SLI dashboards.

**Stakeholder update:** send "change complete" notification within 1 hour of successful deployment.

**Change ticket update:** record actual implementation duration, any deviations, final status.

---

## PIR Scheduling

Before closing the change war room: confirm PIR is scheduled within 5 business days.

Assign a PIR facilitator — not the change owner. The change owner participates in the PIR but does not run it. The conflict of interest is too high for self-facilitation.
