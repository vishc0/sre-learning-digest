## 6. Root Cause of Deviation
*(Complete only if deviations occurred in Section 1. If no deviations, mark N/A.)*

**Deviation category — select all that apply**:

- [ ] **Planning gap** — The RFC was wrong about what would happen. The plan itself was flawed.
- [ ] **Execution gap** — The RFC plan was correct but execution differed from the plan.
- [ ] **Environment gap** — Production behaved differently from staging in a way that could not have been predicted from staging test results.
- [ ] **External dependency gap** — A dependency behaved unexpectedly in a way that the RFC did not account for and the dependency team did not flag.
- [ ] **Tooling gap** — A deployment tool, monitoring system, or rollback mechanism did not behave as expected.

**Root cause description**:
*(For each selected category: describe specifically what happened, what the RFC assumed that turned out to be wrong, and what would need to be true to prevent the same deviation in a future change.)*

**Is this root cause systemic** (likely to affect other teams or other changes, not just this one)?
[ ] Yes — a Problem Record should be opened to track elimination of the systemic cause

## 7. Change Quality Score

Select one. This score is reported in the monthly change metrics dashboard.

- [ ] **Success** — Implemented as planned with no unexpected deviations; customer impact within RFC-estimated bounds; rollback was not required; no post-change incident attributed to this change within 24 hours.

- [ ] **Partial Success** — At least one of: (a) implementation required steps beyond the RFC plan but was completed successfully; (b) window overran RFC estimate by less than 50%; (c) minor post-change issues appeared and were resolved within 24 hours without customer-visible impact; (d) customer impact was within bounds but closer to the upper limit than estimated.

- [ ] **Failure** — At least one of: (a) rollback was required; (b) customer impact exceeded RFC-estimated bounds; (c) a post-change incident was directly attributed to this change; (d) the change could not be completed within the maintenance window; (e) the change was abandoned mid-execution.
